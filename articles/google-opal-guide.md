## はじめに：Opalとの出会い

「プログラミングなしでAIアプリが作れる」と聞いて、最初は半信半疑でした。でも、実際にGoogle Opalを触ってみて、その考えは完全に変わったんです。

ブログ記事のURLを入れるだけで、自動的に内容を要約してSNS投稿文を3パターン生成してくれるアプリ。YouTube動画のサムネイル作成アプリ。こうしたツールが、コードを一行も書かずに、自然な日本語での指示だけで作れてしまう。

これは、AIを使って何かを作りたいと思っているすべての人にとって、大きなチャンスだと感じました。この記事では、そんなGoogle Opalの使い方を、初心者の方にもわかるように解説していきます。

## 第1部：Opalってなに？

「Opal」という名前を初めて聞く方も多いかもしれません。まずは、Opalがどんなツールで、何ができるのかを見ていきましょう。

### 1.1 Opalの基本

**Opalとは？**

Google Opal（オパール）は、Googleの研究開発部門「Google Labs」が2025年7月24日にリリースした実験的なツールです。その最大の特徴は、**プログラミングの知識がなくても、自然な言葉だけでAIを使ったミニアプリが作れる**という点にあります。

従来、AIを活用したアプリを作るには、PythonやJavaScriptなどのプログラミング言語を学ぶ必要がありました。でも、Opalなら「こんなアプリが欲しい」と日本語（または英語）で伝えるだけで、AIがワークフロー（処理の流れ）を自動的に組み立ててくれるんです。

**いつから使えるの？料金は？**

- **リリース日**: 2025年7月24日（当初は米国のみ）
- **日本での利用開始**: 2025年10月7日（15カ国に拡大）
- **料金**: 完全無料（2026年1月現在もベータ版として無料提供中）

無料で使えるというのは、AIツールを試してみたい初心者にとって大きなメリットです。ただし、「実験的なベータ版」であることには注意が必要です（詳しくは後述します）。

### 1.2 Opalの3つの特徴

#### 特徴1：ノーコードで誰でも使える

「ノーコード」というのは、コードを書かずにアプリが作れるという意味です。Opalでは、次のような流れでアプリを作成します：

1. 「こんなアプリが欲しい」と自然な言葉で説明
2. Opalが自動的にワークフロー（視覚的な処理の流れ図）を生成
3. 必要に応じて、ビジュアルエディタで微調整
4. 完成したアプリをすぐに実行・共有

プログラミングの「プ」の字も知らなくて大丈夫。普段使っている言葉で指示を出せば、AIが理解してくれます。

#### 特徴2：GoogleのAIエコシステムと完全統合

Opalの強みは、Googleが提供する最先端のAI技術と連携していることです：

- **Gemini**: Googleの大規模言語モデル（文章生成や理解が得意）
- **Google検索**: Web上の最新情報を取得
- **YouTube**: 動画情報の取得や分析
- **Google Maps**: 地図や場所の情報
- **Imagen**: 画像生成AI
- **Veo**: 動画生成AI

これらのツールを組み合わせることで、テキスト、画像、音声、動画を扱うマルチモーダルなアプリが作れます。

#### 特徴3：簡単に共有できる

作ったアプリは、Googleドキュメントのようにリンクを発行して共有できます。

✅ **良いところ：**
- チームメンバーにリンクを送るだけで使ってもらえる
- 相手はブラウザでアクセスするだけ（インストール不要）
- 権限設定も簡単

❌ **注意点：**
- 現時点では、Webサイトへの埋め込みはできません
- 外部サービスからOpalを呼び出すAPIもまだありません

### 1.3 Opalでできること・できないこと

#### できること

**1. 情報収集と要約**
```
例：「特定のトピックについてGoogle検索して、
     上位5件の記事を要約してまとめて」
```

**2. コンテンツ生成**
```
例：「ブログ記事のタイトルを10個生成して、
     それぞれにYouTube用のサムネイル案も考えて」
```

**3. データ処理と分析**
```
例：「2025年の台風情報をWebで検索して、
     日本語で要約文を作成して」
```

**4. マルチステップのワークフロー**
```
例：「指定したトピックのリサーチ → 構成作成 →
     本文執筆 → 紹介動画の生成まで一括で」
```

#### できないこと・制限事項

正直にお伝えすると、Opalにはいくつかの制限があります：

❌ **外部サービスとの連携が限定的**
- 現状、Google関連のサービス以外との連携は基本的にできません
- OpenAIのGPTモデルや他社のAIサービスを直接呼び出すことはできません
- n8nやZapierのように、あらゆる外部サービスと連携できるわけではありません

❌ **Webサイトへの埋め込みができない**
- 作成したアプリを自社サイトに埋め込む機能はありません
- 外部からOpalを呼び出すためのAPI制御も現時点では存在しません

❌ **UIは英語のみ（2026年1月時点）**
- インターフェース自体は英語表示のみ
- ただし、プロンプト（指示）は日本語でもOKです
- アプリの出力結果も日本語で返してもらえます

⚠️ **ベータ版であることに注意**
- 「実験的なベータ版」のため、サービスが不安定になることも
- 将来的に有料化される可能性があります
- サービス内容の大幅変更や、最悪の場合終了の可能性もゼロではありません

これらの制限を理解した上で、「まずはAIアプリの試作（PoC: Proof of Concept）を作ってみたい」という用途には最適です。

## 第2部：実際に使ってみよう

理論はわかったけど、実際にどうやって使うの？ この第2部では、アカウント登録から、最初のアプリ作成まで、ステップバイステップで解説します。

### 2.1 アカウント登録と初期設定

#### ステップ1: Opalにアクセス

1. ブラウザで https://opal.google/landing/ を開きます
2. 「Get Started」または「Try Opal」ボタンをクリック

#### ステップ2: Googleアカウントでログイン

1. Googleアカウントでログインします（まだない方は作成が必要です）
2. Opalから「Googleドライブへのアクセス許可」を求められます
3. 「許可」をクリック

**なぜGoogleドライブへのアクセスが必要なの？**

作成したOpalアプリは、Googleドライブに保存されます。これは、Googleドキュメントやスプレッドシートと同じような仕組みです。

#### ステップ3: ダッシュボードを確認

ログインに成功すると、Opalのダッシュボード（トップ画面）が表示されます：

- **Create New**: 新しいアプリを作成
- **Recent**: 最近作成・編集したアプリ一覧
- **Templates**: サンプルのテンプレート（参考になります）

### 2.2 はじめてのアプリを作ってみよう

#### 例題：「ブログ記事要約アプリ」を作る

ブログ記事のURLを入力すると、その内容を要約してX（旧Twitter）用の投稿文を3パターン生成してくれるアプリを作ってみましょう。

**ステップ1: 「Create New」をクリック**

ダッシュボードの「Create New」ボタンをクリックします。

**ステップ2: プロンプトを入力**

画面下部にあるプロンプト入力フォームに、作りたいアプリの説明を入力します：

```
ブログ記事のURLを入力すると、その記事の内容を読み取って要約し、
X（旧Twitter）用の告知投稿文を3パターン作成してください。

各投稿文は140文字以内で、以下の要素を含めてください：
- 記事の核心的なポイント
- 読者の興味を引く表現
- 記事へのリンク

3パターンはそれぞれ異なるトーンで作成してください：
1. フォーマルなビジネストーン
2. カジュアルで親しみやすいトーン
3. 疑問形で好奇心を刺激するトーン
```

**日本語でもOK？英語の方がいい？**

どちらでも動きます！私の経験では、日本語で書いても英語で書いても、ワークフローの生成精度は変わりませんでした。ただし、より複雑なアプリを作る場合は、英語の方が微妙なニュアンスが伝わりやすいこともあります。

**ステップ3: ワークフローの自動生成を確認**

プロンプトを送信すると、Opalが自動的にワークフローを作成します：

1. **Input**: ユーザーからブログURLを受け取る
2. **Web Fetch**: そのURLのコンテンツを取得
3. **Gemini（要約）**: 記事内容を要約
4. **Gemini（投稿文生成1）**: ビジネストーンの投稿文を生成
5. **Gemini（投稿文生成2）**: カジュアルトーンの投稿文を生成
6. **Gemini（投稿文生成3）**: 疑問形の投稿文を生成
7. **Output**: 3パターンの投稿文を表示

視覚的には、これらのステップが矢印でつながれた「フローチャート」のように表示されます。

**ステップ4: 試しに実行してみる**

ワークフローが生成されたら、「Run」ボタンをクリックして試してみましょう：

1. ブログ記事のURL（例：あなたのお気に入りのブログ）を入力
2. 「Run」をクリック
3. 数秒〜数十秒で、3パターンの投稿文が生成されます

**ステップ5: 必要に応じて調整**

もし結果が期待と違っていたら？ 大丈夫です。ワークフローの各ステップをクリックして、細かい調整ができます：

- **プロンプトの修正**: 各Geminiステップのプロンプトを編集
- **ステップの追加**: 新しい処理を追加（例：画像生成を追加）
- **ステップの削除**: 不要な処理を削除
- **順序の変更**: ドラッグ&ドロップでステップの順序を変更

### 2.3 ワークフローのビジュアル編集

自動生成されたワークフローは、ビジュアルエディタで自由に編集できます。

#### よく使う編集操作

**1. ステップの追加**

画面上の「+」ボタンをクリックして、新しいステップを追加できます：

- **Gemini**: テキスト生成や理解
- **Search**: Google検索
- **Image generation**: 画像を作成
- **Input**: ユーザーからの入力を受け取る
- **Output**: 結果を表示

**2. ステップの設定を変更**

各ステップをクリックすると、詳細設定パネルが開きます：

```
例：Geminiステップの設定

・プロンプト（指示内容）
・Temperature（創造性の度合い: 0-1）
  0に近い = より正確で一貫した出力
  1に近い = より創造的で多様な出力
・Max tokens（最大出力文字数）
```

**3. 条件分岐の追加**

「もし〜なら、こう処理する」という条件分岐も設定できます：

```
例：記事の文字数によって処理を変える

IF 記事の文字数 > 3000文字
  → 詳細な要約を生成
ELSE
  → 簡潔な要約を生成
```

#### 編集のコツ

**コツ1: 会話形式でも編集できる**

ビジュアルエディタだけでなく、自然な言葉での指示でも編集できます：

```
悪い例：
「画像生成のステップを追加して」
（どこに？どんな画像？）

良い例：
「要約の後に、記事のサムネイル画像を生成するステップを追加してください。
 画像のスタイルは『ミニマルでプロフェッショナル』でお願いします」
```

**コツ2: 一度に一つずつ変更する**

複雑な変更をいきなりすると、どこで問題が起きたか分かりにくくなります。

❌ **良くない進め方：**
```
画像生成も追加して、音声入力も追加して、
言語も5カ国語対応にして...（一気に変更）
```

✅ **良い進め方：**
```
1. まず画像生成を追加 → テスト
2. 問題なければ音声入力を追加 → テスト
3. さらに多言語対応を追加 → テスト
```

**コツ3: こまめに保存する**

Opalは自動保存されますが、大きな変更の前には念のため「Save」をクリックしておきましょう。

### 2.4 作ったアプリを共有しよう

アプリが完成したら、チームメンバーや友人と共有してみましょう。

#### 共有の手順

**ステップ1: 「Share」ボタンをクリック**

画面右上の「Share」ボタンをクリックします。

**ステップ2: 権限を設定**

Googleドキュメントと同じように、権限を選択できます：

- **Viewer（閲覧のみ）**: アプリを実行できるが、編集はできない
- **Editor（編集可能）**: ワークフローを編集できる

**ステップ3: リンクをコピーして送信**

生成されたリンクをコピーして、メールやSlackで送ります。

受け取った人は、リンクをクリックするだけですぐにアプリを使えます。インストールや設定は一切不要です！

#### 共有時の注意点

⚠️ **リンクを知っている人全員がアクセスできる**

「リンクを知っている全員」に共有する設定の場合、リンクが漏れると誰でもアクセスできてしまいます。機密性の高い情報を扱う場合は、特定のメールアドレスだけに共有しましょう。

⚠️ **実行にはOpalアカウントが必要**

共有された人も、Opalにログインする必要があります（Googleアカウントがあれば無料で登録できます）。

## 第3部：実践的な活用例

「Opalでどんなアプリが作れるの？」この疑問に答えるため、実際の活用例をいくつか紹介します。これらはすべて、プログラミングなしで作れるアプリです。

### 3.1 コンテンツ制作者向けアプリ

#### 例1：ブログ記事執筆アシスタント

**何をするアプリ？**

トピックを入力すると、以下を自動生成してくれます：

1. Google検索で最新情報をリサーチ
2. 記事の構成（見出し）を作成
3. 各セクションの本文を執筆
4. 紹介用のYouTubeサムネイル案を生成

**プロンプト例：**

```
以下の機能を持つブログ記事執筆アプリを作成してください：

入力：記事のトピック（例：「Google Opalの使い方」）

処理フロー：
1. トピックに関する最新情報をGoogle検索で収集（上位10件）
2. 検索結果を分析し、記事の構成（見出し5つ）を提案
3. 各見出しに対して、300-500文字の本文を執筆
4. 記事全体のサムネイル画像を生成
   （スタイル：モダンでプロフェッショナル、テキスト入りインフォグラフィック）

出力：
- 記事構成（見出しリスト）
- 各セクションの本文
- サムネイル画像
```

**活用シーン：**
- 初稿作成の時間短縮（アイデア出しから執筆まで）
- トピックのリサーチと最新情報の収集
- サムネイル制作の外注コスト削減

#### 例2：SNS投稿バリエーション生成

**何をするアプリ？**

1つのコンテンツから、複数のSNSプラットフォーム向けの投稿文を一括生成します。

**プロンプト例：**

```
以下の投稿バリエーション生成アプリを作成してください：

入力：
- 元となるコンテンツ（ブログ記事URLまたはテキスト）
- ターゲットプラットフォーム（X, Instagram, LinkedIn, Facebookから選択）

処理フロー：
1. コンテンツを要約して核心ポイントを抽出
2. 各プラットフォーム向けに投稿文を生成：
   - X（旧Twitter）: 140文字、カジュアルなトーン、ハッシュタグ3つ
   - Instagram: キャプション（300文字）、ハッシュタグ10個、絵文字入り
   - LinkedIn: プロフェッショナルなトーン、500文字、業界用語OK
   - Facebook: 親しみやすいトーン、400文字、質問で締める

出力：
各プラットフォーム向けの投稿文（3パターンずつ）
```

**活用シーン：**
- SNSマーケティングの効率化
- 複数プラットフォームへの同時投稿準備
- A/Bテスト用の投稿バリエーション作成

### 3.2 ビジネス・業務効率化アプリ

#### 例3：議事録作成＆タスク管理ツール

**何をするアプリ？**

会議の音声入力（または書き起こしテキスト）から、議事録とタスクリストを自動生成します。

**プロンプト例：**

```
会議の書き起こしテキストを入力すると、以下を生成するアプリを作成してください：

入力：会議の書き起こしテキスト

処理フロー：
1. 書き起こしテキストを分析
2. 議事録を構造化されたフォーマットで作成：
   - 日時・参加者
   - 議題ごとの要点
   - 決定事項
   - 保留事項
3. タスクリストを抽出：
   - タスク内容
   - 担当者（発言から推測）
   - 期限（言及されていた場合）
   - 優先度（高/中/低）

出力：
- 構造化された議事録（Markdown形式）
- タスクリスト（チェックボックス付き）
```

**活用シーン：**
- 会議後の議事録作成時間を大幅削減
- タスクの抜け漏れ防止
- 誰が何をいつまでにやるかの明確化

#### 例4：採用業務効率化ツール

**何をするアプリ？**

応募者の履歴書から、スキルマッチング分析と面接質問を自動生成します。

**プロンプト例：**

```
以下の採用支援アプリを作成してください：

入力：
- 応募者の履歴書（テキストまたはPDF）
- 募集職種の要件定義

処理フロー：
1. 履歴書から主要スキルと経験を抽出
2. 募集要件とのマッチング分析（スコアリング）：
   - 必須スキル：各項目について○/△/×で評価
   - 優遇スキル：該当する項目をリストアップ
   - 総合スコア：100点満点で算出
3. 面接質問を生成：
   - 履歴書の内容を深掘りする質問（5つ）
   - スキルを確認するための質問（5つ）
   - カルチャーフィットを見る質問（3つ）

出力：
- マッチング分析レポート
- 推奨面接質問リスト
```

**活用シーン：**
- 大量の応募者の一次スクリーニング
- 面接官への事前資料作成
- 採用基準の標準化

### 3.3 リサーチ・情報収集アプリ

#### 例5：競合分析ツール

**何をするアプリ？**

競合企業名を入力すると、Web上の情報を収集して分析レポートを作成します。

**プロンプト例：**

```
競合企業の情報収集・分析アプリを作成してください：

入力：
- 競合企業名（最大5社）
- 分析の観点（例：製品特徴、価格戦略、マーケティング手法）

処理フロー：
1. 各企業についてGoogle検索（企業サイト、ニュース記事、レビューなど）
2. 以下の情報を抽出：
   - 主力製品・サービス
   - 価格帯
   - ターゲット顧客層
   - 強みと弱み
   - 最近のニュース・動向
3. 比較分析レポートを作成：
   - 表形式での比較（企業 vs 各項目）
   - SWOT分析（各企業）
   - 市場でのポジショニング

出力：
- 競合分析レポート（Markdown形式）
- 比較表
```

**活用シーン：**
- 新規市場参入前の競合調査
- 自社の強み・弱みの相対的な把握
- マーケティング戦略の立案

## 第4部：他のツールとの比較

「Opalって、他のノーコードツールと何が違うの？」という疑問に答えるため、よく比較される「n8n」と「Dify」との違いを見ていきます。

正直に言うと、私も最初はこの3つのツールの違いがよくわからず、いろいろ試しながら理解していきました。それぞれに強みがあるので、目的に応じて使い分けることが大切です。

### 4.1 Opal vs n8n vs Dify：比較表

まずは、3つのツールの主な違いを表で確認しましょう：

| 項目 | Opal | n8n | Dify |
|------|------|-----|------|
| **開発元** | Google Labs | n8n GmbH | Dify.AI |
| **料金** | 完全無料（ベータ版） | 無料版あり<br>有料版: $20〜/月 | 無料版あり<br>有料版: $59〜/月 |
| **対象ユーザー** | AI初心者〜中級者 | 技術者〜中級者 | AI/LLM開発者 |
| **主な用途** | AIアプリの迅速な試作 | 業務自動化・統合 | 高度なAIアプリ開発 |
| **学習難易度** | ★☆☆（簡単） | ★★☆（中程度） | ★★★（やや難しい） |
| **AI連携** | Google AIのみ | 多数のAIサービス対応 | 複数のLLM対応 |
| **外部連携** | Google系サービス中心 | 400以上のアプリ連携 | API経由で柔軟 |
| **セルフホスト** | ❌ 不可 | ✅ 可能 | ✅ 可能 |
| **コード編集** | ❌ 不可 | ✅ JavaScript/Python可 | △ 一部可 |
| **API提供** | ❌ なし（2026年1月時点） | ✅ あり | ✅ あり |

### 4.2 各ツールの詳細比較

#### Opal：最もシンプルで直感的

**強み：**

✅ **学習コストが最小**
- 自然言語で指示するだけでアプリが作れる
- プログラミング知識が一切不要
- 90分以内にPoCが作れるスピード感

✅ **Google AIエコシステムとの統合**
- Gemini（テキスト生成）
- Imagen（画像生成）
- Veo（動画生成）
- すべて追加料金なしで利用可能

✅ **完全無料**
- ベータ版の間は、すべての機能を無料で使える

**弱み：**

❌ **連携の制限**
- Google以外のサービスとの連携が限定的
- OpenAIのGPTモデルなどは直接使えない

❌ **カスタマイズの制限**
- コードを書いて細かい制御をすることはできない
- 複雑なロジックの実装は難しい

❌ **エンタープライズ機能の欠如**
- ユーザー管理機能が弱い
- 監査ログなどのガバナンス機能がない

**向いている人・用途：**
- AIアプリを初めて作る人
- アイデアを素早く形にしたい人
- Googleのサービスをメインで使っている人
- PoC（概念実証）を短時間で作りたい人

#### n8n：最も柔軟で技術的

**強み：**

✅ **圧倒的な連携力**
- 400以上の外部アプリと連携可能（Slack、Notion、Shopify、Salesforce など）
- Webhook、HTTP Request、カスタムAPIも自由に使える

✅ **完全なセルフホスト対応**
- 自社サーバーで運用できる
- データを完全に自社管理できる（セキュリティ重視の企業に最適）

✅ **コード編集が可能**
- JavaScript/Pythonコードを埋め込める
- 複雑なデータ変換やロジックを実装できる

**弱み：**

❌ **学習コストが高い**
- ノード（処理ブロック）の組み合わせ方に慣れが必要
- APIの仕組みやHTTPリクエストの知識が求められる場面も

❌ **AI特化ではない**
- AIツールの一つとして使えるが、AI開発に特化しているわけではない

**向いている人・用途：**
- 業務プロセス全体を自動化したい人
- 複数のSaaSツールをつなげたい人
- データを自社で管理したい企業
- エンジニアリングチーム

#### Dify：AI/LLM開発に特化

**強み：**

✅ **高度なAI機能**
- RAG（検索拡張生成）パイプラインの構築
- 自律的なエージェント機能
- マルチモーダル（テキスト・画像・音声）対応

✅ **複数のLLMを選択可能**
- OpenAI GPT、Anthropic Claude、Google Gemini など
- モデルごとの切り替えが簡単
- コスト最適化のためのモデル選択が可能

✅ **プロンプト管理機能**
- バージョン管理
- A/Bテスト
- プロンプトの再利用とテンプレート化

**弱み：**

❌ **学習コストが高い**
- RAG、エージェント、プロンプトエンジニアリングなどの知識が必要
- AI/LLM開発の経験がないと使いこなすのが難しい

❌ **料金が高め**
- 無料版は機能制限あり
- 本格的に使うには有料プラン（$59〜/月）が必要

**向いている人・用途：**
- 高度なAIアプリケーションを開発したい人
- RAGやエージェントを活用したい人
- 複数のLLMを比較・活用したい開発チーム
- プロダクション環境でのAIアプリ運用

### 4.3 どのツールを選ぶべき？

ここまで読んで、「結局、どれを使えばいいの？」と思った方へ。私のおすすめは、**段階的に使い分ける**ことです。

#### 推奨される段階的アプローチ

```

このフェーズで分かること：

- 外部サービス（Slack、Salesforce など）との連携が必要

- 複数のLLMを使い分けたい

```

- 最初からn8nで作っていたら、学習コストで時間がかかっていたはず
- 段階的アプローチのおかげで、効率的に目的を達成できた

## まとめ：これからOpalを始めるあなたへ

長い記事でしたが、お疲れさまでした！ここまで読んでくださって、ありがとうございます。

Google Opalは、「AIアプリを作ってみたいけど、プログラミングは難しそう」と思っている人にとって、最高の入り口です。自然な言葉で指示するだけで、数分〜数十分でアプリが作れてしまう体験は、本当に感動的です。

### Opalを始める前に覚えておいてほしいこと

**1. 完璧を目指さなくていい**

最初から複雑なアプリを作る必要はありません。まずは簡単なアプリから始めて、徐々に機能を追加していきましょう。失敗しても大丈夫、何度でもやり直せます。

**2. ベータ版であることを理解する**

Opalはまだ「実験的なベータ版」です。将来的に有料化されたり、サービスが変更されたりする可能性があります。重要な業務に使う前に、その点を理解しておきましょう。

**3. 他のツールとの組み合わせを考える**

Opalだけですべてを解決しようとせず、n8nやDifyなど他のツールとの組み合わせも検討してみてください。それぞれの強みを活かせば、もっと強力なアプリが作れます。

### これからの学び方

**ステップ1: とにかく触ってみる**

この記事を読んだら、すぐにOpalにアクセスして、簡単なアプリを作ってみてください。「百聞は一見にしかず」です。

**ステップ2: テンプレートから学ぶ**

Opalのダッシュボードには、サンプルテンプレートがあります。これらを開いて、どんなワークフローになっているかを見るだけでも勉強になります。

**ステップ3: コミュニティで情報交換**

X（旧Twitter）やRedditなどで「#GoogleOpal」と検索すると、他のユーザーが作ったアプリや活用事例が見つかります。そこから学ぶことも多いです。

**ステップ4: 小さく始めて、大きく育てる**

最初は個人用の簡単なアプリから。慣れてきたら、チーム向けのアプリ、そして業務で使えるアプリへと段階的にレベルアップしていきましょう。

### 最後に

AIツールの世界は、日々進化しています。Google Opalも、これからどんどん新しい機能が追加されていくでしょう。

大切なのは、「完璧なツール」を待つのではなく、「今あるツール」で何ができるかを試してみることです。そして、試行錯誤を楽しみながら、少しずつ自分のスキルを伸ばしていくことです。

この記事が、あなたのAIアプリ作成の旅の第一歩になれば、とてもうれしいです。さあ、早速Opalを開いて、最初のアプリを作ってみましょう！
