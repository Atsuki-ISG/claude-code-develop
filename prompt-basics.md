# プロンプトの基礎

## プロンプトとは

プロンプト（Prompt）とは、AI言語モデルに対して指示や質問を与えるためのテキスト入力のことです。プロンプトの質によって、AIからの応答の質が大きく変わります。効果的なプロンプトを作成することを「プロンプトエンジニアリング」と呼びます。

## プロンプトの基本構造

良いプロンプトには、以下の要素が含まれることが多いです：

### 1. 役割の設定
AIにどのような役割や専門性を持って応答してほしいかを明確にします。

**例：**
```
あなたは経験豊富なプログラミング講師です。
```

### 2. タスクの明確化
具体的に何をしてほしいのかを明確に伝えます。

**例：**
```
初心者向けにPythonの変数について説明してください。
```

### 3. コンテキストの提供
必要な背景情報や制約条件を提供します。

**例：**
```
対象は中学生で、プログラミング経験はありません。
```

### 4. 出力形式の指定
どのような形式で回答してほしいかを指定します。

**例：**
```
箇条書きで3つのポイントにまとめてください。
```

## 良いプロンプトを書くための原則

### 明確性
あいまいな表現を避け、具体的に指示を出します。

❌ **悪い例：**
```
プログラミングについて教えて
```

✅ **良い例：**
```
Python言語における関数の定義方法と、関数を使うメリットを3つ教えてください。
```

### 具体性
抽象的な質問ではなく、具体的な例を含めます。

❌ **悪い例：**
```
コードを改善して
```

✅ **良い例：**
```
以下のPythonコードのパフォーマンスを改善してください。特にループ処理を最適化し、
実行時間を短縮する方法を提案してください。

[コードを記載]
```

### 段階的な指示
複雑なタスクは段階的に分解して指示します。

**例：**
```
以下の手順でWebアプリケーションの設計を支援してください：
1. まず、主要な機能要件を整理してください
2. 次に、必要なデータベーステーブルを設計してください
3. 最後に、APIエンドポイントの一覧を提案してください
```

## プロンプトのパターン

### 質問形式
情報を得たいときに使用します。

```
機械学習における過学習とは何ですか？その対策方法も教えてください。
```

### 指示形式
特定のタスクを実行してほしいときに使用します。

```
以下のテキストを英語に翻訳してください：
「本日は晴天なり」
```

### 例示形式（Few-shot Learning）
期待する出力の例を示してから、新しいタスクを依頼します。

```
以下の例に従って、文章を要約してください。

例1:
入力: 「今日は天気が良かったので、公園に行って散歩をしました。桜がとても綺麗でした。」
出力: 「公園で散歩、桜が綺麗」

例2:
入力: 「昨日の会議では新製品の開発について議論しました。予算と期限が主な論点でした。」
出力: 「新製品開発会議、予算と期限を議論」

それでは以下を要約してください：
入力: 「先週末に友人と山登りに行きました。頂上からの景色は素晴らしく、疲れも吹き飛びました。」
```

### 制約付き形式
特定の条件や制約を設けます。

```
100文字以内で、量子コンピュータの基本概念を説明してください。
```

## よくある落とし穴と対策

### 落とし穴1: 前提が不明確
AIがあなたの知識レベルや目的を理解していない場合、的外れな回答になることがあります。

**対策：** 自分の立場や目的を明示する
```
私はWeb開発初心者です。Reactを学び始めたばかりですが、
「コンポーネント」という概念がよく理解できません。
具体例を使って説明してください。
```

### 落とし穴2: 一度に多くを求めすぎる
複数の複雑なタスクを一つのプロンプトに詰め込むと、回答の質が下がります。

**対策：** タスクを分割して段階的に進める

### 落とし穴3: 出力形式が曖昧
「詳しく説明して」だけでは、どの程度の詳しさが必要か伝わりません。

**対策：** 具体的な形式を指定する
```
以下の形式で回答してください：
- 概要: 2-3行
- メリット: 箇条書きで3つ
- デメリット: 箇条書きで3つ
- 具体例: 1つ
```

## プロンプトエンジニアリングのベストプラクティス

1. **反復的な改善**: 最初のプロンプトで完璧な結果が得られなくても、フィードバックを元に改善していきます。

2. **明示的な制約**: 文字数、形式、トーン、専門用語の使用など、制約は明確に伝えます。

3. **文脈の保持**: 会話型のやり取りでは、前の回答を参照しながら段階的に深掘りします。

4. **具体例の活用**: 期待する出力の例を示すことで、より正確な結果が得られます。

5. **検証可能な質問**: 回答が検証可能な形で得られるよう、具体的な質問をします。

## まとめ

効果的なプロンプトを作成するには：

- **明確で具体的な指示**を出す
- **必要な文脈と制約**を提供する
- **期待する出力形式**を明示する
- **段階的なアプローチ**を取る
- **反復的に改善**する

プロンプトエンジニアリングは、実践を通じて上達していくスキルです。様々なパターンを試しながら、自分なりの効果的なプロンプト作成方法を見つけていきましょう。
